# feynman — 费曼输出文档

## 何时触发

用户说"费曼输出"、"教学输出"、"feynman"、"讲给别人听"、"输出教学"。

也可以指定材料：`/feynman lecun1989` 或 `/feynman 全部最近`。

## ⚠️ 不可违反的规则

**禁止调用 `skill_manage`。**

## Vault 路径

```bash
VAULT="${ESPACE_VAULT:-$HOME/Espace_Obsidian}"
```

## 执行流程

### 第 1 步：选择材料

如果没有指定材料，扫描最近 7 天新摄入的内容，列出让用户选。

如果指定了 slug 或标题，读取对应的：
- `原始材料/文章/<slug>.md`（原文）
- `知识库/摘要/<slug>-摘要.md`（已有摘要）
- `知识库/概念/` 中相关的概念页

### 第 2 步：检查是否已有费曼输出

```bash
ls "$VAULT/输出/教学/" 2>/dev/null
```

如果 `<slug>-费曼输出.md` 已存在，回复用户：

```
📖 费曼输出已存在: 输出/教学/<slug>-费曼输出.md
如需重新生成，请回复「覆盖」
```

等待用户确认后再覆盖。如果是 cron 自动运行（无用户交互），跳过此步骤。

### 第 3 步：生成费曼输出文档（通过确认后）

保存到 `$VAULT/输出/教学/<slug>-费曼输出.md`

```markdown
---
title: "原标题 — 费曼输出"
基于: "原始材料/文章/<slug>.md"
创建日期: "今天"
类型: 教学输出
---

# 费曼输出: 原标题

## 🎯 一句话总结

<!-- 用一句话让完全不懂的人也听懂 -->

## ❓ 这篇文章在讲什么问题？

<!-- 核心问题是什么 -->

## 🔧 作者怎么解决的？

<!-- 方法和思路 -->

## 💡 我学到了什么

<!-- 你的核心收获 -->

## 🗣️ 如何讲给新人听

<!-- 用比喻/类比/生活例子解释核心概念，让零基础的人听懂 -->

## 🤔 Q&A

<!-- 预判读者可能问的问题并提前回答 -->

**Q:**
**A:**

## 🔗 关键概念

- [[概念名]] → 一句话解释

## 📚 延伸阅读

<!-- 如果想深入，推荐看什么 -->

## ✅ 我的行动项

<!-- 这篇文章对我有什么实际启发或行动 -->
```

### 第 3 步：git push 并回复

```bash
cd "$VAULT" && git add -A && git commit -m "教学输出: 原标题" && git push
```

回复：

```
📖 费曼输出已生成
━━━━━━━━━━━━━━
📄 输出/教学/<slug>-费曼输出.md
📝 这是一篇「讲给别人听」级别的输出文档
🔄 已推送 GitHub

要点预览：
🎯 一句话...
❓ 核心问题...
💡 学到...
```

## 输出原则

- **用最简单的语言** — 假设读者完全不懂这个领域
- **多用类比** — "这就像..."
- **自问自答** — 预判困惑点
- **不要复制原文** — 用自己的话重组
- **长度不限** — 该长则长，该短则短，关键是让人看懂
